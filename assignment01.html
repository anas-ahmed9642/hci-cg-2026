Semantic Comparison: Wrong vs. Correct

In HCI, semantic HTML reduces the Bridge of Execution. When the browser understands the structure, the user can more easily form an action plan (Norman's Model).

1. Document Structure (Norman's Evaluation)

Proper structure allows users and screen readers to perceive the information hierarchy accurately. (Ref: Dix et al., Ch. 10 "Universal Design", p. 374)

WRONG: Using visual styling to fake a heading.

KLM Model Analysis

CORRECT: Using real heading tags for assistive technology.

KLM Model Analysis

2. Form Controls (KLM Mental Preparation)

Using labels reduces the M (Mental) operator time for the user by clearly defining input targets. (Ref: Dix et al., Ch. 12 "Cognitive Models", p. 428)

WRONG: Placeholder only / No label.

Email:

CORRECT: Using the <label> tag.

3. Site Navigation

Semantic navigation blocks help users conceptualize the site structure, aiding in wayfinding. (Ref: Dix et al., Ch. 7 "Design Rules", p. 275)

WRONG: Using generic containers.

Home | CV | Contact

CORRECT: Using the <nav> element.

  • Home
  • CV
  • Contact

4. Accessibility (Norman's Perception)

Missing alternative text breaks the perception phase for visually impaired users. (Ref: Dix et al., Ch. 10 "Universal Design", p. 376)

WRONG: Missing Alternative Text.

CORRECT: Descriptive alt text for screen readers.

5. Clickable Elements (Affordances)

Using proper interactive elements provides clear affordances, reducing cognitive load. (Ref: Dix et al., Ch. 3 "The Interaction", p. 126-128)

WRONG: Using a div with an onClick event.

Submit

CORRECT: Using the semantic button element.

6. Emphasis and Importance (Semantic Meaning)

Visual-only tags fail to convey importance to screen readers, disrupting evaluation. (Ref: Dix et al., Ch. 1 "The Human", p. 15-18)

WRONG: Using bold and italic tags strictly for style.

Note: Do not click twice.

CORRECT: Using emphasis and strong tags for semantic weight.

Note: Do not click twice.

7. Page Titles (Goal Formulation)

A clear title helps users verify they are in the right system state, aiding in goal formulation. (Ref: Dix et al., Ch. 3 "The Interaction", p. 128)

WRONG: Leaving the document untitled or generic.

<title>Document</title>

CORRECT: Descriptive titles outlining the page's purpose.

<title>Lab 01 - Semantic HTML & HCI Models</title>

8. Table Structure (System State Perception)

Proper table headers allow assistive technologies to map data logically. (Ref: Dix et al., Ch. 12 "Cognitive Models", p. 423)

WRONG: Using standard rows for headers.

<tr><td>Name</td><td>Age</td></tr>

CORRECT: Using the <th> element within a <thead>.

<thead><tr><th>Name</th><th>Age</th></tr></thead>

9. Main Content Area (KLM Selection Rules)

Wrapping core content in a main tag reduces navigation time for screen readers, optimizing selection. (Ref: Dix et al., Ch. 12 "Cognitive Models", p. 428-430)

WRONG: A generic wrapper for the primary content.

<div id="content">...</div>

CORRECT: Using the semantic main tag.

<main>...</main>

10. Quotations (Attribution Context)

Using blockquotes preserves context and meaning, mapping directly to the user's mental model. (Ref: Dix et al., Ch. 7 "Design Rules", p. 260)

WRONG: Using paragraph tags with quotation marks.

<p>"Design is how it works." - Steve Jobs</p>

CORRECT: Using blockquote and cite tags.

<blockquote> <p>Design is how it works.</p> <cite>- Steve Jobs</cite> </blockquote>
