Semantic Comparison: Wrong vs. Correct

In HCI, semantic HTML reduces the Bridge of Execution. When the browser understands the structure, the user can more easily form an action plan based on standard Cognitive Models.

1. Document Structure (Cognitive Walkthrough)

Proper structure ensures the user's next action is visible and perceivable, a core question in a Cognitive Walkthrough. (Ref: Dix et al., Ch. 9 "Evaluation techniques")

WRONG: Using visual styling to fake a heading.

KLM Model Analysis

CORRECT: Using real heading tags for assistive technology.

KLM Model Analysis

2. Form Controls (KLM Mental Preparation)

Using labels reduces the M (Mental) operator time for the user by clearly defining input targets. (Ref: Dix et al., Ch. 12 "Cognitive models")

WRONG: Placeholder only / No label.

CORRECT: Using the <label> tag.

3. Site Navigation (GOMS Methods)

Semantic navigation blocks help users execute standard Methods in the GOMS model for moving between pages. (Ref: Dix et al., Ch. 12 "Cognitive models")

WRONG: Using generic containers.

CORRECT: Using the <nav> element.

4. Accessibility (Heuristic Evaluation)

Missing alternative text violates heuristic evaluation standards for system visibility and universal access. (Ref: Dix et al., Ch. 9 "Evaluation techniques")

WRONG: Missing Alternative Text.

CORRECT: Descriptive alt text for screen readers.

5. Clickable Elements (KLM Pointing Operator)

Clear, semantic interactive elements optimize the P (Pointing) operator in Keystroke-Level Modeling. (Ref: Dix et al., Ch. 12 "Cognitive models")

CORRECT: Using the semantic button element.
